# Discord Bot Request - Alternative Messages (Non-Trading Focus)

---

## **Option 1: Personal Notification System** (Most Casual)

```
Hi @Admin,

I'm working on a personal notification system to make sure I don't miss important alerts from the trading channels (#stock-alerts, #day-trade-alerts, #swings).

I travel frequently and sometimes Discord notifications don't come through reliably on mobile. Would it be okay to add a read-only bot that monitors those channels and sends me custom notifications to my phone?

Bot would just:
- Read messages from those 3 channels
- Send me push notifications when new charts/setups are posted
- Completely passive (no posting, no access to other channels)

Basically a more reliable notification system since I don't want to miss time-sensitive setups.

Let me know if that works!
```

## **Recommended Approach by Server Culture:**

### **Casual/Friendly Server:**
Use **Option 1 (Notification System)** or **Option 6 (Alert Filtering)**
- Simple, relatable problem (missing notifications)
- Low-key request
- Easy to approve

### **Professional/Serious Trading Server:**
Use **Option 2 (Research/Analysis)** or **Option 5 (Study Tool)**
- Shows you're serious about learning
- Data-driven approach
- Aligns with trading education

### **Tech-Savvy Server:**
Use **Option 4 (Custom App)**
- Technical users understand custom tools
- Building better UX is relatable
- Shows initiative

### **Conservative/Cautious Admin:**
Use **Option 3 (Organization/Archive)**
- Least "advanced" sounding
- Simple use case
- Low perceived risk
